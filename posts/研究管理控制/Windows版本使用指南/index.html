Windows版研究项目管理脚本安装使用指南

概述

本指南提供了Linux研究管理脚本的Windows版本，包含两个版本：

  • PowerShell版本 (resman.ps1) - 功能完整，推荐使用
  • 批处理版本 (resman.bat) - 兼容性好，适合传统Windows用户

系统要求

基本要求

  • Windows 10/11 或 Windows Server 2016+
  • PowerShell 5.1+ (Windows 10/11自带)
  • Git for Windows

推荐配置

  • Windows Terminal (现代终端体验)
  • PowerShell 7+ (跨平台PowerShell)

安装步骤

1. 安装Git for Windows

  1. 访问 Git官网 下载Git for Windows

  2. 运行安装程序，建议选择以下选项：

    • 选择"Git from the command line and also from 3rd-party software"
    • 选择"Use Windows’ default console window"
    • 其他选项保持默认

  3. 验证安装：

2. 配置Git

3. 下载脚本文件

将以下文件下载到您的计算机：

  • resman.ps1 (PowerShell版本)
  • resman.bat (批处理版本)

4. 设置脚本位置

方法1：放在用户目录

方法2：添加到系统PATH

  1. 将脚本放在任意目录（如 C:\Tools\）
  2. 右键"此电脑" → “属性” → “高级系统设置”
  3. 点击"环境变量"
  4. 在"系统变量"中找到"Path"，点击"编辑"
  5. 添加脚本所在目录路径

2. Git命令不识别

错误信息：'git' 不是内部或外部命令

解决方法：

  1. 确认Git for Windows已正确安装
  2. 重启命令提示符或PowerShell
  3. 检查PATH环境变量是否包含Git路径

4. 备份功能失败

可能原因：

  • 磁盘空间不足
  • 权限不够
  • 文件被占用

解决方法：

  • 检查磁盘空间
  • 以管理员身份运行
  • 关闭占用文件的程序

与Linux版本的差异

特性 Linux版本 Windows版本 说明
脚本语言 Bash PowerShell/Batch 语法差异
路径分隔符 / \ 自动处理
颜色支持 ANSI转义码 PowerShell颜色 显示效果相同
文件权限 chmod ACL Windows使用不同权限模型
符号链接 ln -s mklink Windows需要管理员权限
备份工具 rsync robocopy 功能相似
